# Code of Conduct

## Our Pledge

Mailbridge MCP is a security- and privacy-sensitive project. We expect project discussions to stay respectful, factual, and focused on improving the software.

Participants should:

- Treat others with respect, even during disagreement.
- Keep technical feedback specific and actionable.
- Avoid harassment, insults, threats, or discriminatory language.
- Avoid posting private data, credentials, access tokens, mail content, logs with secrets, or personal information.
- Report security issues privately instead of publishing exploit details in public issues.

## Scope

This code applies to project spaces including GitHub issues, pull requests, discussions, release comments, and any maintainer-run communication channels.

## Security And Privacy Expectations

Because Mailbridge handles mail accounts, MCP bearer tokens, contact data, calendar data, and message content, contributors must not request or publish:

- Real passwords, app passwords, OAuth tokens, or MCP bearer tokens.
- Private mail bodies, address books, calendars, or audit logs containing personal data.
- Full production databases or `./data` directories.
- Public proof-of-concept exploit steps before coordinated disclosure.

Use redacted logs and synthetic examples whenever possible.

## Enforcement

Maintainers may edit, hide, or remove comments and issues that violate this code. Repeated or severe violations may result in temporary or permanent blocking from the project.

## Attribution

This code is adapted for the Mailbridge MCP project and follows the intent of the Contributor Covenant without copying it verbatim.
